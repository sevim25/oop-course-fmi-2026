# Iterator

Итераторът е обект, който предоставя **унифициран начин за достъп до елементи в дадена последователност**.

Интуитивно може да се мисли като „указател“:

- може да сочи към елемент
- можем да го преместим към следващ елемент
- често можем да прочетем стойността чрез `*it`
- понякога можем и да пишем през него

Това не означава, че всеки итератор е буквално указател. По-скоро той се държи като абстракция за позиция в контейнер.

Една от основните идеи е да се отделят:

- **данните** — контейнерите
- **достъпът до данните** — итераторите
- **операциите върху данните** — алгоритмите

Така един и същ алгоритъм може да работи върху различни контейнери, стига те да предоставят подходящи итератори.

Например `std::find` не се интересува дали обхожда `std::vector`, `std::string` или `std::list`. Той работи с диапазон от итератори.

---

## Основна идея: диапазон `[begin, end)`

Много алгоритми в STL приемат два итератора:

```cpp
begin, end
```

Това описва диапазона:

```text
[begin, end)
```

Тоест:

- `begin` сочи към първия елемент в диапазона;
- `end` сочи **един след последния** елемент.

Пример:

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {10, 20, 30};

    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << ' ';
    }
}
```

Тук:

- `v.begin()` сочи към `10`;
- `v.end()` не сочи към валиден елемент за dereference, а сочи след последния
- `v.end()` служи като граничен маркер

---

## Основни операции върху итератор

Най-често се използват следните операции:

- `*it` — достъп до елемента
- `++it` — преминаване към следващата позиция
- `it1 == it2`, `it1 != it2` — сравнение
- при някои итератори: `--it`, `it + n`, `it - n`, `it[n]`

Пример:

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3};

    auto it = v.begin();
    std::cout << *it << '\n'; // 1

    ++it;
    std::cout << *it << '\n'; // 2
}
```

## Iterator Design Pattern

В по-общия смисъл **Iterator design pattern** е дизайн идея, при която обхождането на елементите се отделя от самата структура от данни.

Тоест клиентът не трябва да знае вътрешното представяне на колекцията. Достатъчно е да има механизъм който да позволява:

- вземи начало
- провери дали сме стигнали края
- премини нататък
- прочети текущия елемент

STL реализира тази идея много успешно чрез итератори и алгоритми.

---

## Пример с различни контейнери

### `std::vector`

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3};

    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << ' ';
    }
}
```

### `std::string`

```cpp
#include <iostream>
#include <string>

int main() {
    std::string s = "abc";

    for (auto it = s.begin(); it != s.end(); ++it) {
        std::cout << *it << ' ';
    }
}
```

Идеята е една и съща: алгоритъмът не зависи от конкретния контейнер, а от интерфейса на итератора.

---

## Const iterators

Ако не искаме да променяме елементите, често използваме `const_iterator` или `cbegin()` / `cend()`.

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3};

    for (auto it = v.cbegin(); it != v.cend(); ++it) {
        std::cout << *it << ' ';
    }
}
```

Тук можем да четем, но не и да променяме елементите през итератора.

---

## Reverse iterators

Можем да обхождаме и наобратно чрез `rbegin()` и `rend()`.

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3, 4};

    for (auto it = v.rbegin(); it != v.rend(); ++it) {
        std::cout << *it << ' ';
    }
}
```

---

## Range-based for и итератори

Когато пишем:

```cpp
for (int x : v) {
    // ...
}
```

компилаторът концептуално използва `begin()` и `end()`.

Тоест range-based `for` е удобен синтаксис върху същата фундаментална идея за итератори.

---

## Защо итераторите са по-добри от работа по индекс в много случаи

Работа по индекс е удобна при `vector` и `string`, но не всички контейнери имат смислен индекс.

Например при по-комплексни структури като `std::list` няма произволен достъп по индекс. Итераторите дават общ интерфейс, който работи с много различни структури.

Така кодът става:

- по-универсален
- по-съвместим със STL
- по-лесен за комбиниране със стандартните алгоритми

---

## Итератори и алгоритми

Итераторите са в основата на STL алгоритмичните функции.

Пример:

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {7, 1, 9, 3};

    auto it = std::find(v.begin(), v.end(), 9);
    if (it != v.end()) {
        std::cout << "Found: " << *it << '\n';
    }
}
```

Алгоритъмът `std::find` не е длъжен да знае нищо за `vector` като структура. Той просто получава диапазон.

---

## Инвалидация на итератори

Това е много важна практическа тема.

При някои операции итераторите могат да станат невалидни. Това означава, че след промяна в контейнера старият итератор вече не трябва да се използва.

### Пример с `vector`

При добавяне на нов елемент е възможно да се направи реалокация на паметта. Тогава всички стари итератори, указатели и референции към елементите могат да станат невалидни.

```cpp
#include <vector>

int main() {
    std::vector<int> v = {1, 2, 3};
    auto it = v.begin();

    v.push_back(4); // may invalidate it

    // *it; // potentially invalid
}
```

Това е една от най-честите практически грешки.

### Общ принцип

Когато модифицираме контейнер, трябва да знаем дали:

- размерът се променя
- паметта се реалокира
- дадената операция инвалидира итератори

---

## Разлика между iterator и pointer

Итераторът често прилича на указател, но не е задължително да бъде реален raw pointer.

При `vector` и `string` итераторите често са близки по поведение до указатели. При други контейнери това не е така.

Затова е по-добре да ги мислим като абстракция за „позиция в диапазон“, а не просто като `T*`.

---

## Пример: ръчно обхождане и алгоритъм

### Ръчно обхождане

```cpp
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {2, 4, 6};

    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << ' ';
    }
}
```

### С алгоритъм

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v = {2, 4, 6};

    std::for_each(v.begin(), v.end(), [](int x) {
        std::cout << x << ' ';
    });
}
```

И в двата случая итераторите са механизмът, чрез който достигаме до елементите.

---

## Обобщение:

1. Итераторът е абстракция за позиция в контейнер.
2. Обикновено работим с диапазони от вида `[begin, end)`.
3. `end()` е маркер след последния елемент и не се dereference-ва
4. Повечето от STL алгоритмите работят с итератори, не директно с контейнери.
5. Не всички итератори имат еднакви възможности.
6. При модификация на контейнерите трябва да внимаваме за инвалидация на итератори.

Итераторите позволяват:
- еднакъв начин за обхождане на различни контейнери
- отделяне между структура на данните и алгоритъм
